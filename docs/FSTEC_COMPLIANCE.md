# PhantomProxy — Соответствие требованиям ФСТЭК

## Цель

Обеспечить возможность сертификации PhantomProxy по требованиям ФСТЭК России (УЗ-1, УЗ-2) для использования в госструктурах и критичной инфраструктуре.

## Ключевые требования

### 1. Шифрование (ГОСТ)

| Компонент | Текущее | Целевое |
|-----------|---------|---------|
| TLS | OpenSSL (AES, RSA) | Криптопровайдер КриптоПро с ГОСТ Р 34.10-2012, ГОСТ Р 34.11-2012 |
| БД | SQLite (no encryption) | PostgreSQL + прозрачное шифрование ТД (ГОСТ) |
| Логи | Plain text / JSON | Шифрование логов по ГОСТ Р 34.12-2015 |

### 2. Identity & Access (Keycloak / Zitadel)

- Поддержка ФСТЭК УЗ-1/УЗ-2: ГОСТ-сертифицированные СКЗИ
- mTLS между сервисами
- Роли: admin, operator, analyst, read-only

### 3. Аудит

- Все действия с учётными данными и сессиями — в audit log
- Retention по политике (например, 1 год)
- Защита audit log от изменений

### 4. On-Prem / Self-Hosted

- Полный развёртывание без внешних облаков
- Локальный LLM (Llama, GigaChat) — без отправки данных в облако

## Roadmap

1. **Phase 1** — Интеграция с КриптоПро CSP (опциональный TLS)
2. **Phase 2** — Шифрование БД по ГОСТ
3. **Phase 3** — Keycloak/Zitadel с российской сертификацией
